# Fix: 404 Error on /gas-proxy

You're seeing:
```
GET http://localhost:5173/gas-proxy?action=init... → 404 Not Found
```

## Quick Fix (3 Steps)

### 1. Verify VITE_GAS_URL is Set

Check your `.env` file:

```bash
cat .env | grep VITE_GAS_URL
```

Should output something like:
```
VITE_GAS_URL=https://script.google.com/macros/s/AKfycbwK...xyz.../exec
```

**Not empty or missing!**

If it's empty → Run `./deploy.sh` to deploy GAS backend and get the URL.

### 2. Restart Dev Server

The environment variables are loaded at startup. After changing `.env`:

```bash
# Stop the dev server (Ctrl+C)
npm run dev
```

### 3. Test the URL

In browser, visit your GAS URL directly:

```
https://script.google.com/macros/s/{DEPLOYMENT_ID}/exec?action=init
```

You should see JSON response like:
```json
{
  "ok": true,
  "data": {
    "months": [...],
    "budget": {...},
    "openingBal": {...}
  }
}
```

If you get HTML or login page → See "GAS Access Restricted" section below.

## Detailed Solutions

### Issue: VITE_GAS_URL is Empty

**Fix:**

```bash
# Deploy the GAS backend
./deploy.sh

# This will automatically update .env with the deployment URL
# Then restart dev server
npm run dev
```

### Issue: GAS Returns HTML (Access Restricted)

**Symptoms:**
- Browser shows HTML page or login redirect
- Not JSON response

**Fix:**

1. Go to https://script.google.com
2. Click your project: "FinTracker"
3. Click **Deploy** → **Manage deployments**
4. Click the **edit** icon (pencil) next to your deployment
5. Find **"Who has access"** dropdown
6. Change to **"Anyone"**
7. Click **Update**
8. Refresh http://localhost:5173/fintracker/

### Issue: Request Never Reaches GAS (404 in Browser)

**The Vite proxy isn't intercepting /gas-proxy requests**

**Check:**

1. Did you restart dev server after changing `.env`? (required!)
2. Is the request actually going to `/gas-proxy`? (F12 → Network tab)
3. Is there an error in Vite console?

**Fix:**

```bash
# Kill the dev server
npm run dev

# Watch for startup messages. Should include:
# → API_URL (from env)
# → Proxy configured to GAS_URL

# If no proxy message, VITE_GAS_URL might be empty
```

### Issue: CORS Error ("blocked by CORS policy")

**Symptoms:**
- Request reaches GAS but browser blocks response
- Console shows `Access to XMLHttpRequest blocked...`

**Fix:**

1. Verify GAS deployment allows "Anyone":
   - Go to script.google.com → FinTracker → Deploy → Manage
   - Edit deployment, set "Who has access" to "Anyone"
   - Update

2. Clear browser cache:
   - F12 → Application → Clear storage
   - Or hard refresh (Cmd+Shift+R on Mac, Ctrl+Shift+R on Windows)

3. Try incognito window (fresh cache, no extensions)

## Verify Setup

Run this to check your setup:

```bash
# 1. Check environment
echo "=== Environment ==="
grep "VITE_" .env | head -3

# 2. Check GAS URL is valid
echo "=== Testing GAS URL ==="
GAS_URL=$(grep VITE_GAS_URL .env | cut -d= -f2)
if [ ! -z "$GAS_URL" ]; then
  curl -s "$GAS_URL?action=init" | head -c 100
  echo ""
else
  echo "ERROR: VITE_GAS_URL is empty!"
fi

# 3. Check if dev server is running
echo "=== Dev Server ==="
curl -s http://localhost:5173 | head -c 50
```

## Still Not Working?

1. **Check browser console (F12)**
   - Look for red error messages
   - Check Network tab for the failed request

2. **Check Vite console**
   - Any error messages during startup?
   - Does it show the proxy configuration?

3. **Check the actual error response**
   - F12 → Network tab
   - Click the failed `/gas-proxy` request
   - Go to Response tab
   - What does it show?

4. **Try direct URL**
   - Paste your VITE_GAS_URL directly in browser
   - Does it work?
   - Do you get JSON or error?

## Emergency Fix

If you need to bypass the proxy during development:

1. Open `.env`
2. Add: `VITE_API_URL=https://script.google.com/macros/s/{YOUR_DEPLOYMENT_ID}/exec`
3. Remove or comment out: `VITE_GAS_URL=...`
4. Restart: `npm run dev`

This directly uses the GAS URL instead of going through the proxy.
